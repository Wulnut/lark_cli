## SearchGroup 筛选组合

筛选器中一行是一个 SearchParam（一行中的筛选参数、操作符、筛选值组成一个 SearchParam），一组为一个 SearchGroup，具体筛选组合的用法可参考下文「常用查询示例」。

![SearchGroup示意图](https://p9-arcosite.byteimg.com/tos-cn-i-goo7wpa0wc/06ec5fd2da4e4dc78a9a01d6ab0a5686~tplv-goo7wpa0wc-quality:q75.awebp)

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| search_params | list<SearchParam> | 筛选条件列表，包括筛选参数、操作符、筛选值 |
| conjunction | string | 组合关系，枚举 AND/OR，对应筛选器中的「且」和「或」 |
| search_groups | list<SearchGroup> | 嵌套的筛选组合 |

## SearchParam 筛选参数

> **说明**：筛选参数不支持字段类型为公式计算的字段。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| param_key | string | 筛选条件参数 key，普通字段为 field_key，特殊字段为固定参数 key（如 current_nodes） |
| value | interface{} | 搜索字段值，普通字段按字段值格式，特殊字段参考固定参数说明 |
| operator | string | 操作符类型，枚举值见下文「操作符枚举值」 |
| pre_operator | string | 前置操作符，目前仅用于复合字段搜索场景 |
| value_search_groups | SearchGroup | 复合字段父字段且 operator 为 MEET/NOT MEET 时使用 |

## 操作符枚举值

| 操作符 | 说明 |
| --- | --- |
| `~` | 匹配 |
| `!~` | 不匹配 |
| `=` | 等于 |
| `!=` | 不等于 |
| `<` | 小于 |
| `>` | 大于 |
| `<=` | 小于等于 |
| `>=` | 大于等于 |
| `HAS ANY OF` | 存在选项属于 |
| `HAS NONE OF` | 不存在选项属于 |
| `IS NULL` | 为空 |
| `IS NOT NULL` | 不为空 |
| `CONTAINS` | 包含 |
| `NOT CONTAINS` | 不包含 |
| `MEET` | 满足 |
| `NOT MEET` | 不满足 |

## 前置操作符枚举值

| 前置操作符 | 说明 |
| --- | --- |
| `EVERY` | 每一组 |
| `ANY` | 存在一组 |

## 固定参数及取值

> **说明**：如果操作符选的是「为空」或「不为空」，不校验参数值（除流程节点时间、节点负责人、角色人员会校验参数值），只校验参数是否支持该操作符。

| 参数名 | 参数 key | 操作符可取值 | 参数值类型 | 参数值示例 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 进行中节点 | current_nodes | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`CONTAINS`、`NOT CONTAINS` | list<string> | `["开发中", "测试中"]` | 节点名称列表，可从「获取流程模板配置详情」中获取 |
| 流程节点 | all_states | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`CONTAINS`、`NOT CONTAINS` | list<string> | `["待开始", "进行中", "已完成"]` | 节点名称列表 |
| 流程节点时间 | feature_state_time | `<`、`>`、`<=`、`>=`、`IS NULL`、`IS NOT NULL` | object | 见下方示例 | - |
| 全部人员 | people | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL`、`CONTAINS`、`NOT CONTAINS` | list<string> | `["user_key_1", "user_key_2"]` | user_key 列表 |
| 创建时间 | created_at | `=`、`!=`、`<`、`>`、`<=`、`>=` | int64 | `1702310399000` | 毫秒时间戳 |
| 节点负责人 | node_owners | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL`、`CONTAINS`、`NOT CONTAINS` | list<object> | 见下方示例 | - |
| 工作项 ID | work_item_id | `=`、`!=`、`<`、`>`、`<=`、`>=`、`HAS ANY OF`、`HAS NONE OF` | list<int64> | `[12345, 45678]` | - |
| 工作项状态 | work_item_status | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF` | list<string> | `["start", "doing", "end"]` | 从「获取字段信息」接口获取 options |
| 模板 ID | template_id | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL` | list<int64> | `[123, 456]` | 通过「获取工作项下的流程模板列表」获取 |
| 业务线 | business | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL` | list<string> | `["business_id_1"]` | 从「获取空间下业务线详情」获取 |
| 角色人员 | role_owners | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL`、`CONTAINS`、`NOT CONTAINS` | list<object> | 见下方示例 | - |

### 流程节点时间参数格式

```json
{
  "state_name": "Android开发估分",      // 节点名称
  "state_timestamp": 1702310399000,     // 节点筛选时间戳（毫秒）
  "state_condition": 1                 // 筛选状态：0=开始，1=结束
}
```

### 节点负责人参数格式

```json
[
  {
    "state_name": "开始",               // 节点名称
    "owners": ["user_key_1"]           // 负责人 user_key 列表
  },
  {
    "state_name": "进行中",
    "owners": ["user_key_2"]
  }
]
```

### 角色人员参数格式

```json
[
  {
    "role": "pm",                      // 角色 ID
    "owners": ["user_key_1"]           // user_key 列表
  }
]
```

## 自定义类型及取值

若筛选条件是普通字段则 param_key 为字段 key（field_key），value 为该字段值的格式。

> **说明**：如果操作符选的是「为空/不为空」，不校验字段值，只校验当前字段类型是否支持该操作符。

| 参数类型 | 字段类型 Key | 操作符可取值 | 字段值类型 | 说明 |
| --- | --- | --- | --- | --- |
| 单行文本/多行文本 | text | `~`、`!~`、`=`、`!=`、`IS NULL`、`IS NOT NULL` | string | 匹配使用前后模糊匹配 |
| 数字 | number | `=`、`!=`、`<`、`>`、`<=`、`>=`、`IS NULL`、`IS NOT NULL` | float64 | - |
| 链接 | link | `~`、`!~`、`=`、`!=`、`IS NULL`、`IS NOT NULL` | string | 匹配链接字符串 |
| 开关 | bool | `=`、`!=`、`IS NULL`、`IS NOT NULL` | bool | 搜索空值需使用 `IS NULL` |
| 单值系统外信号 | signal | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF` | list<string> | `"undefined"`=暂无，`"null"`=进行中，`"false"`=未通过，`"true"`=已通过 |
| 单选 | select | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL` | list<string> | 使用选项 value 值（非 label） |
| 单选按钮 | radio | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL` | list<string> | - |
| 多选 | multi-select | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL`、`CONTAINS`、`NOT CONTAINS` | list<string> | - |
| 级联单选 | tree-select | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL` | list<string> | - |
| 级联多选 | tree-multi-select | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL` | list<string> | - |
| 单选人员 | user | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL` | list<string> | user_key 列表 |
| 多选人员 | multi-user | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL`、`CONTAINS`、`NOT CONTAINS` | list<string> | user_key 列表 |
| 富文本 | multi-text | `~`、`!~`、`IS NULL`、`IS NOT NULL` | string | 仅无格式值匹配 |
| 单选关联工作项 | workitem_related_select | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL` | list<int64> | 关联工作项 ID 列表 |
| 多选关联工作项 | workitem_related_multi_select | `=`、`!=`、`HAS ANY OF`、`HAS NONE OF`、`IS NULL`、`IS NOT NULL`、`CONTAINS`、`NOT CONTAINS` | list<int64> | 关联工作项 ID 列表 |
| 日期时间 | precise_date | `<`、`>`、`<=`、`>=`、`IS NULL`、`IS NOT NULL` | int64 | 毫秒时间戳 |
| 日期 | date | `<`、`>`、`<=`、`>=`、`IS NULL`、`IS NOT NULL` | int64 | 毫秒时间戳 |
| 复合字段-父字段 | compound_field | `IS NULL`、`IS NOT NULL`、`MEET`、`NOT MEET` | - | 需配合 pre_operator 和 value_search_groups 使用 |
| 复合字段-子字段 | - | 支持上述除复合字段外的所有操作符 | - | 直接填写时支持 pre_operator，嵌套时不支持 |
| 电话号码 | telephone | `~`、`!~`、`=`、`!=`、`IS NULL`、`IS NOT NULL` | string | - |
| 电子邮件 | email | `~`、`!~`、`=`、`!=`、`IS NULL`、`IS NOT NULL` | string | - |

## 常用查询示例

### 查询需求下指定状态，且创建时间在某个区间的工作项列表

```json
{
  "pagination": {
    "page_size": 50,
    "page_num": 1
  },
  "data_sources": [
    {
      "project_key": "6645776axxxxxxb1fb2b9ef8",
      "work_item_type_keys": "story"
    }
  ],
  "search_group": {
    "conjunction": "AND",
    "search_params": [
      {
        "param_key": "created_at",
        "value": 1654064482000,
        "operator": ">"
      },
      {
        "param_key": "created_at",
        "value": 1654063482000,
        "operator": "<"
      },
      {
        "param_key": "work_item_status",
        "value": ["start"],
        "operator": "="
      }
    ]
  }
}
```

### 复合字段数据 - 每一组子字段都满足条件

使用 `pre_operator: EVERY` 配合 `operator: MEET`：

```json
{
  "pagination": {
    "page_size": 50,
    "page_num": 1
  },
  "data_sources": [
    {
      "project_key": "6645776axxxxxxb1fb2b9ef8",
      "work_item_type_keys": "story"
    }
  ],
  "search_group": {
    "conjunction": "AND",
    "search_params": [
      {
        "param_key": "field_ba8f2e",
        "operator": "MEET",
        "pre_operator": "EVERY",
        "value_search_groups": {
          "conjunction": "AND",
          "search_params": [
            {
              "param_key": "field_4396f8",
              "operator": "=",
              "value": "1"
            }
          ]
```

### 复合字段数据 - 存在一组子字段满足条件

```json

    "page_num": 1

    {
      "project_key": "6645776axxxxxxb1fb2b9ef8",

    }
  ],
  "search_group": {
    "conjunction": "AND",
    "search_params": [
      {
        "param_key": "field_ba8f2e",
        "operator": "MEET",
        "pre_operator": "ANY",
        "value_search_groups": {
          "conjunction": "AND",
          "search_params": [
            {
              "param_key": "field_4396f8",
              "operator": "=",
              "value": "1"
            }

        }
      }
    ]
  }
}
```
