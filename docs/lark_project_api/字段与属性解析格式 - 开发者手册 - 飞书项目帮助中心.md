Open API 统一了用户创建和查询的字段格式，以属性和字段的方式进行描述。

- **属性**：作为工作项的原始信息展示，不以 field 的方式进行暴露，主要是平台-字段管理页面的系统字段以及内置基础字段。
- **字段**：区别于原始信息，作为通用的 field 方式使用，平台-字段管理页面的部分系统字段和用户自定义字段。

## 属性（Properties）

属性是工作项的系统字段，作为原始信息展示。

| 属性类型 | API 字段名 | 数据类型 | 说明 |
| --- | --- | --- | --- |
| 工作项 ID | id | int64 | 出参：工作项唯一标识 |
| 工作项名称 | name | string | 出参：工作项名称 |
| 工作项状态 | work_item_status | object | 出参/入参：状态对象 |
| 工作项模板 ID | template_id | int64 | 出参/入参：模板 ID |
| 工作项模板类型 | template_type | string | 出参：模板类型 |
| 空间 Key | project_key | string | 出参：所属空间 Key |
| 工作项类型 | work_item_type_key | string | 出参：工作项类型 Key |
| 工作项模式 | pattern | string | 出参：Node（节点流）/ State（状态流） |
| 当前节点 | current_nodes | array | 出参：当前所在节点列表 |
| 创建者 | created_by | string | 出参：创建者 user_key |
| 更新者 | updated_by | string | 出参：更新者 user_key |
| 删除者 | deleted_by | string | 出参：删除者 user_key |
| 创建时间 | created_at | int64 | 出参：毫秒时间戳 |
| 更新时间 | updated_at | int64 | 出参：毫秒时间戳 |
| 删除时间 | deleted_at | int64 | 出参：毫秒时间戳 |

### 工作项状态格式

**出参格式：**

```json
{
  "state_key": "doing",
  "is_archived_state": true,
  "is_init_state": false,
  "updated_at": 1721960280400,
  "updated_by": "73567952xxxx8297476"
}
```

**入参格式：**

```json
{
  "state_key": "doing"
}
```

```json
    "id": "doing",

  }

```

## 字段（Fields）

字段是自定义字段或部分系统字段，通过 field_key 进行访问。

### 字段结构

所有字段都包含以下基础结构：

```json
{
  "field_key": "field_xxx",
  "field_type_key": "text",
  "field_alias": "",
  "field_value": "字段值"
}
```

### 单行文本 / 多行文本

| 类型 | field_type_key | 入参类型 | 出参类型 |
| --- | --- | --- | --- |
| 单行文本 | text | string | string |
| 多行文本 | text | string | string |

**示例：**

```json
{
  "field_key": "field_a2697f",
  "field_type_key": "text",
  "field_alias": "",
  "field_value": "单行文本"
}
```

### 富文本

| 类型 | field_type_key | 入参类型 | 出参类型 |
| --- | --- | --- | --- |
| 富文本 | multi_text | array | string / object |

> **注意**：若在 OpenAPI 写入富文本时，直接按照查询结果入参，可能导致视图、度量、自动化等场景不可用。请严格参照「富文本格式」章节进行传参。

**入参格式（结构体数组）：**

```json
{
  "field_key": "field_1e9078",
  "field_value": [
    {
      "type": "paragraph",
      "content": [
        {
          "type": "text",
          "text": "文本",
          "attrs": {
            "fontColor": "blue",
            "italic": "true",
            "underline": "true"
          }
        }
      ]
    }
  ],
  "field_type_key": "multi_text",
  "field_alias": ""
}
```

**出参格式（简单内容）：**

```json
{
  "field_key": "field_1e9078",
  "field_value": "文本\n",
  "field_type_key": "multi_text",
  "field_alias": ""
}
```

## 富文本格式

富文本可应用于富文本字段及评论。

> **重要说明**：若在 OpenAPI 写入富文本时，直接按照查询结果入参，所写入的值能够在实例详情页正常展示。
>
> 然而，视图、度量、自动化等场景依赖 doc_html、doc、doc_text 间满足特定转换规则。请在入参时严格参照以下传参规则。

### 传参规则

| Field | Value |
| --- | --- |
| type | paragraph（文本段落）、blank（空白行）、checklist（待办列表）、ul（无序列表）、ol（有序列表）、horizontalLine（分割线）、table（表格） |
| content | 行内容数组 |
| children.type | text（文本）、hyperlink（超链接）、img（图片）、linkPreview（链接解析） |
| children.text | 文本内容 |
| children.attrs | backgroundColor、bold、fontColor、fontSize、italic、strikethrough、underline |
| lineAttrs | indent（缩进数）、align、blockquote |

### 颜色列表

| Color | RGB |
| --- | --- |
| black | rgb(26, 26, 26) |
| white | rgb(255, 255, 255) |
| blue | rgb(76, 136, 255) |
| green | rgb(84, 194, 72) |
| purple | rgb(127, 59, 245) |
| yellow | rgb(250, 200, 35) |
| red | rgb(240, 91, 86) |

### 富文本分类

- **叶子结点**：hyperlink、image、linkPreview
- **行结构**：align、blockquote、indent
- **行内容**：blank、checklist、horizontalLine、ol、ul、table
- **文字属性**：backgroundColor、bold、fontColor、fontSize、italic、textDecoration、text

## 常见 FAQ

### 关注人（watchers）字段属于什么类型

关注人属于多选人员（multi_user），类似字段可以参考字段管理页面的字段类型。

### 如何更新富文本字段文本加图片格式

1. 通过 Open API「添加附件」在富文本字段中增加图片
2. 更新富文本字段，按照上述「富文本格式」示例进行传递

### 评论支持富文本吗

是的，新增/更新评论 API 支持富文本。使用 `body` 体的 key `rich_text`：

```json
{
  "rich_text": [
    {
      "type": "paragraph",
      "content": [
        {
          "type": "text",
          "text": "editor sdk"
        }
      ]
    }
  ]
}
```
